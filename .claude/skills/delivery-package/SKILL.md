# 納品パッケージスキル

納品物一式を検収可能な状態に整える。`docs/90-pack/deliverables-catalog.md`（成果物カタログ）を正とし、documentation-specialist と各横断エージェントに最終チェックを委譲する。

## 手順

### 1. 納品範囲の確認
- 対象工程（中間納品/最終納品）と契約上の納品物リストをユーザーに確認する
- 契約書・注文書レベルの解釈が必要な場合は判断せずエスカレーションする

### 2. 突合（ギャップ検出）
`docs/90-pack/deliverables-catalog.md` の該当工程の成果物リストと実ファイルを突合し、ギャップ表を作る:

| 成果物 | カタログ要求 | 実在 | 版数 | 承認記録 | 判定 |
|---|---|---|---|---|---|

判定: OK / 欠落 / 未承認 / 版数不整合 / 体裁不備

### 3. 品質ゲート（並列で委譲）
- quality-performance: テスト完了条件・品質メトリクスの最終確認
- documentation-specialist: **仮定マーカー（`仮定: Q-xxx`）の残存検査** — 未クローズの仮定が納品物に残る場合は「既知の制限事項」として顧客提示文書に明記する（黙って納品しない）
- security-compliance: セキュリティチェックリスト・秘密情報混入チェック（納品物内のAPIキー・内部URL・個人情報の検索）
- data-model-specialist: ERD三者一致監査（`/erd-update` の整合性監査）
- documentation-specialist: 全文書の版数・変更履歴・レビュー記録欄・日付整合

### 4. 納品物一覧の生成
`deliverables/<納品日>/納品物一覧.md` を作成:
- 納品物名 / ファイル名 / 版数 / 作成日 / 検収基準との対応（どのACを満たす証跡か）
- 媒体・形式（顧客指定: Excel変換要否など）はCLAUDE.mdの顧客標準に従う。Excel納品が必要な成果物は `/excel-deliverables` で変換し、検品チェックリストを通す

### 5. 最終報告
ユーザーに以下のみ報告する:
- ギャップ表の要旨（欠落・不備の件数と内容）
- 品質ゲートの合否
- 納品可否の判定案（最終判断は人間）

## 原則

- **欠落を勝手に埋めない**。存在しない成果物をこの段階で捏造的に生成するのは禁止。欠落として報告し、作成するかはユーザーが決める
- 顧客に出す前のファイルからは、内部メモ・TODO・AIとのやりとりの痕跡を除去する
- 納品後の変更は版数を上げて変更履歴に残す（上書き納品禁止）
