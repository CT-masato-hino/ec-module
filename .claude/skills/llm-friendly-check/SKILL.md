# LLMフレンドリー診断スキル（人間側の週次チェック)

`/context-health-check` がAIのコンテキスト汚染を診るのに対し、このスキルは**人間の使い方とリポジトリの状態**を診る。
AIの出力品質が落ちる原因の半分は入力側（指示・リポジトリ）にある、が前提。

出力: `docs/90-pack/standards/llm-friendly-report/YYYYMMDD.md`（スコアと改善提案）

## A. リポジトリの機械診断（AIが自動実行）

- [ ] **巨大ファイル**: 1,500行超のソース・500行超のドキュメントを列挙（LLMが一度に読めない正本は分割候補）
- [ ] **CLAUDE.mdの鮮度**: 最終更新日と、記載内容の実態乖離（技術スタック・テーラリング記録が現実と合っているか）
- [ ] **工程正本の鮮度**: `docs/10-management/project-phase.md` の current_phase・ゲート判定記録が実際の作業内容（直近のコミットtype・成果物の出現場所）と合っているか（乖離＝黙った越境のサイン）
- [ ] **正本の重複**: 同じ情報が2箇所に書かれていないか（差分が出た瞬間にAIがどちらを信じるか問題になる）
- [ ] **用語ゆれ**: 機能一覧・用語集・コードの識別子で同一概念に複数の名前がないか
- [ ] **「読めば分かる」の実在**: エージェントが参照する正本（規約・QCD基準・ERD）が実在し、TBD・空欄で放置されていないか
- [ ] **ディレクトリの予測可能性**: 新しい成果物の置き場所が deliverables-catalog の定義から一意に決まるか

## B. 指示の質セルフチェック（人間が3分で回答。今週の自分の依頼を思い出して）

| # | 質問 | LLMフレンドリーな状態 |
|---|---|---|
| 1 | 依頼にゴールと完了条件を言ったか | 「XをYの状態にして。完了条件はZ」で始めている |
| 2 | 1依頼1テーマだったか | 「ついでにあれも」を同一セッションに混ぜていない |
| 3 | AIの確認質問に答えたか | 「いい感じで」で流さず、判断材料を渡している |
| 4 | 全文貼りをしていないか | ログ・仕様書はパスを渡すか要約を渡している（コンテキスト汚染は人間も起こす） |
| 5 | 途中割り込みの頻度 | 方針転換はセッションを切ってから（1セッション内での方針3転は赤信号） |
| 6 | 結果を検収したか | AIの完了報告を読まずに次を頼んでいない（未検収の積み上げは仮定の塔と同じ） |
| 7 | 決定を外部化したか | 会話の中だけで決めたことを docs/ に残した（または残させた） |

## C. 運用習慣の点検（記録から確認）

- [ ] セッション長: 1テーマを超えて延命したセッションの有無（/context-history の更新履歴から推定）
- [ ] エスカレーション対応速度: AIが「人間の判断が必要」と止めた案件の放置日数（止まったAIは工数を生まない — D-05と同じ扱い）
- [ ] ガードレール衝動: 権限確認を面倒がってバイパスしたくなった場面の有無（あったら ai-security-baseline の設定見直し対象。我慢は統制ではない）

## 出力形式

```markdown
# LLMフレンドリー診断 YYYY-MM-DD
## スコア: リポジトリ x/6 ・ 指示の質 y/7 ・ 運用習慣 z/3
## 今週の改善提案（1つだけ）
（最も効果が大きいもの1つに絞る。全部直せという診断は誰も直さない）
## 機械診断の検出明細（ファイルパスつき）
## 先週の改善提案のフォロー（実行されたか。されていなければ提案を変えるか捨てる）
```

## 日次の軽量版（このスキルは呼ばない。CLAUDE.mdの運用ルール）

セッション終了時に自分に5問だけ（30秒）:
1. このセッションの決定は docs/ に残ったか
2. 次のセッションの自分は、何も覚えていなくても再開できるか
3. AIに「察してもらった」箇所はないか（あれば明文化して渡し直す）
4. 未コミット変更を残していないか（意図して残す場合は理由をLATEST.mdに明記）
5. マージ済みブランチ・不要worktreeを離れて畳んだか

## 原則

- 診断はスコアで人を責めるためではなく、**入力側の改善が出力品質に直結する**ことを可視化するため
- 改善提案は週1つ。守られない週が2回続いたら提案の粒度が大きすぎる（分割する）
- 結果は ai-dev-standardizer の振り返り（週次）へ入力し、組織的な傾向（全員が同じ躓き方をしている等）はパック/教育へ還元する
