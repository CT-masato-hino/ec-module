# 競合比較(本モジュール / BASE / Shopify)

## 1. 前提

本ドキュメントは、コーポレートサイトに小規模ECを組み込む際の導入判断の材料として、本モジュール(以下「本モジュール」)とBASE・Shopifyという代表的なASP型ECサービスを事実ベースで比較したものである。マーケティング的な誇張や他社を貶める表現はせず、公開情報に基づいて記載する。

- **比較時点**: 2026年7月
- 手数料・料金プランは各社の都合により変動する。本ドキュメントの数値は執筆時点のものであり、**導入前に必ず各社公式サイトで最新情報を確認すること**(出典は末尾に記載)
- 本モジュールは自社運用のオープンソース構成(Cloudflare Pages + D1 + Stripe)であり、BASE・Shopifyのような単一ベンダーのSaaSではない。比較表は機能・費用面の参考情報であり、運用体制や技術的な前提が異なる点に留意する

## 2. 3者比較表

| 項目 | 本モジュール | BASE(スタンダード/グロース) | Shopify(ベーシック) |
|---|---|---|---|
| 初期費用 | 0円 | 0円 | 0円 |
| 月額固定費 | 0円(Cloudflare無料枠内の場合) | スタンダード: 0円 / グロース: 16,580円(年払) | 3,650円(年払)〜4,850円(月払) |
| 決済・販売手数料 | Stripe決済手数料3.6%のみ | スタンダード: 決済3.6%+40円/件+サービス料3% / グロース: 2.9% | Shopifyペイメント3.55%(国内カード) |
| 決済手段 | カード決済(Stripe)+銀行振込。Stripe対応分(コンビニ払い等)にも対応可。**PayPay・キャリア決済は非対応** | かんたん決済で7種類の決済手段に対応(クレジットカード・コンビニ・キャリア決済・PayPay等) | Shopifyペイメント+決済アプリでの拡張 |
| 独自ドメイン | 対応(自社インフラのため制約なし) | 対応(プランにより条件あり) | 対応 |
| デザイン自由度 | コード直接編集のため自由度は最大。ただしHTML/CSS/TypeScriptの知識が前提 | テーマ内でのカスタマイズが中心 | テーマ+Liquidテンプレートでのカスタマイズ |
| データ所有権 | 自社のD1データベース・R2ストレージに保存。エクスポート・移行は自由 | BASEのプラットフォーム上に保存 | Shopifyのプラットフォーム上に保存 |
| 拡張方法 | コード変更(AIコーディングエージェント含む) | 公式アプリ・テーマの追加 | 公式/サードパーティアプリの追加 |

手数料体系は各社サイトを参照(出典セクション参照)。BASEはスタンダード(無料プラン)とグロース(月額プラン)の2種類があり、本表はスタンダードを基準に比較する。

## 3. 月商別コストシミュレーション

**計算前提**:
- 平均客単価を5,000円と仮定し、月商から注文件数を算出する(月商10万円=20件、30万円=60件、100万円=200件)
- 本モジュール: Stripe決済手数料3.6%のみ、固定費0円
- BASEスタンダード: 「決済手数料3.6%+40円/件」+「サービス料3%」を合算した実質6.6%+40円/件で概算
- Shopifyベーシック: 年払月額3,650円+Shopifyペイメント3.55%(国内カード)で概算。月払(4,850円)の場合はさらにコストが増える
- いずれも送料・消費税・返金・チャージバック等は考慮しない概算値であり、実際の負担額は各サービスの契約条件により変動する

| 月商 | 本モジュール(手数料) | BASEスタンダード(手数料) | Shopifyベーシック(月額+手数料) |
|---|---|---|---|
| 10万円(20件) | 3,600円 | 7,400円(6.6%+40円×20件) | 7,200円(3,650円+3,550円) |
| 30万円(60件) | 10,800円 | 22,200円(6.6%+40円×60件) | 14,300円(3,650円+10,650円) |
| 100万円(200件) | 36,000円 | 74,000円(6.6%+40円×200件) | 39,150円(3,650円+35,500円) |

いずれの月商帯でも、固定費がなくStripe手数料のみで済む本モジュールが試算上は最も低コストになる。ただし本モジュールは自社でインフラ・コードを保守する前提であり、BASE・Shopifyはその保守をベンダーが担う対価として手数料・月額費用を負担する構造である点は公平のため付記する。

## 4. 本モジュールが適するケース / 適さないケース

### 適するケース

- コーポレートサイトに付随する小規模EC(商品点数〜100点程度、国内向け、決済手段はカード決済+銀行振込で足りる)
- 決済手数料を含む運用コストを最小化したい
- デザイン・データを完全に自社で所有・コントロールしたい

### 適さないケース

- PayPay・キャリア決済など、Stripe以外の決済手段が必須
- 月商が数百万円規模まで成長している、または成長を前提とした多機能要件がある(在庫連携・複数店舗・高度な分析等)→ Shopify等の検討が現実的
- 越境EC、実店舗POS連携、サブスクリプション課金が必要
- コードに触れず自分たちだけで完結して運用したい → BASE等のノーコードASPが向く

## 5. AI駆動保守という構造的差別化

本モジュール最大の差別化要因は、月額費用の低さそのものよりも「AIコーディングエージェントによる改修・保守が構造的にしやすい」という点にある。

- BASE・Shopifyはクローズドなプラットフォームであり、AIエージェントに渡せる情報はテーマファイルやアプリ設定など断片的なものに限られる。一方、本モジュールはコード・DBスキーマ・インフラ定義・運用手順(`AGENTS.md`)までリポジトリ一式がそのままAIコーディングエージェントのコンテキストに収まる
- 実際のAPIキーを使わないモック決済・モックメール(`email_logs`テーブルへの記録)と、ローカルで完結するD1/R2により、AIエージェントは外部サービスへの依存なしに「実装→動作確認→テストデータの掃除」までを完結できる
- 自動テスト(Vitest 67件)とE2Eスモークテスト、それを検証するCI(`.github/workflows/ci.yml`)が回帰の門番として機能するため、AIエージェントによる改修を人が都度全量確認しなくても一定の安全性を持って受け入れられる
- 同梱の開発ハーネス: 設計ルール・検証手順をまとめた`AGENTS.md`、定型作業のスラッシュコマンド(`/reset-demo`・`/clear-sample-data`・`/integrate-corporate-site`)、環境初期化コマンド(`npm run init`)、Webhook動作確認スクリプト(`scripts/send-test-webhook.mjs`)

この結果、「改修・カスタマイズの限界費用が低い」ことが、月額固定費0円と並ぶ本モジュールの価値になっている。ただし、これはAIエージェントの利用費用や人による最終確認が不要になるという意味ではない。AIエージェントの実行にはその利用料が別途かかり、生成された変更を本番に反映する前には人間による最終確認が必要であることは正直に付記しておく。
